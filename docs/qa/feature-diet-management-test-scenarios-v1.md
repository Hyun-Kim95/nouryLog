# 식단 관리 기능 QA 시나리오 v1

## 1) 앱 핵심 시나리오
- [ ] 음식 사진 업로드 -> OCR 성공 -> 사용자 확정 -> 저장 성공
- [ ] OCR 일부 필드 누락 -> 수동 보완 후 저장 성공
- [ ] OCR 실패(외부 API 장애) -> 수동 입력 전환 -> 저장 성공
- [ ] OCR 무료 5회 소진 전까지 정상 사용 가능 확인
- [ ] OCR 5회 초과 + 미결제 상태에서 결제 유도 화면 노출 확인
- [ ] OCR 결제 후 5회 초과 상태에서도 사용 가능 확인
- [ ] 음수 값 입력 -> 저장 차단 + 필드 오류 메시지 노출
- [ ] 동일 저장 연타 -> 중복 생성 없이 1건만 저장

## 2) 통계 시나리오
- [ ] 하루/주/월 전환 시 총합/달성률 정상 노출
- [ ] 배치 지연(`isStale=true`) 시 지연 배너 노출
- [ ] 데이터 없음 시 빈 상태 + 기록 입력 CTA 노출
- [ ] 타임존 변경 후 기준 타임존 표시 확인

## 3) 관리자 시나리오
- [ ] 회원 목록 필터/검색/페이지네이션(15개) 동작
- [ ] 회원 비활성화 후 로그인 차단 확인
- [ ] 음식 템플릿 비활성화 후 신규 기록 선택 불가 확인
- [ ] 문의 상태 변경(접수->처리중->완료) 동작
- [ ] 공지 작성/수정/비활성화 동작
- [ ] 통계 지연 6시간 초과 시 대시보드 경고 노출 확인
- [ ] `최신값 반영(강제 재집계)` 실행 후 상태 갱신 확인

## 4) 삭제/보존 시나리오
- [ ] 비활성화 데이터 기본 조회 제외 확인
- [ ] includeInactive 조회에서만 노출 확인
- [ ] 이미 비활성화된 대상 재요청 시 멱등 성공 확인
- [ ] `deactivatedAt + 1년` 경과 데이터 purge 배치 처리 확인

## 5) 오류/권한 시나리오
- [ ] 401 발생 시 재인증 흐름 동작
- [ ] 403 발생 시 접근 불가 안내 노출
- [ ] 5xx/타임아웃 시 재시도 UX 동작
- [ ] 오류 응답 `traceId` 로그 추적 가능 여부 확인
- [ ] `402 PAYMENT_REQUIRED` 발생 시 결제 플로우 연결 확인

## 6) 광고/과금 시나리오
- [ ] 기본 플랜 사용자 하단 배너 광고 노출 확인
- [ ] `premium_monthly` 결제 후 하단 광고 제거 확인
- [ ] 결제 복구(restore purchase) 후 OCR 유료 권한/광고 제거 복원 확인
- [ ] 단일 SKU(`premium_monthly`) 결제 시 OCR 유료권 + 광고 제거가 함께 활성화되는지 확인
- [ ] OCR 4회 사용 시 사전 안내 배너 노출 확인
- [ ] OCR 5회 소진 시 결제 유도 모달 문구가 정책 문구와 일치하는지 확인
- [ ] entitlement 응답의 `nextPaywallTrigger` 값이 사용량 구간(`ocr_remaining_1`, `ocr_exhausted`)과 일치하는지 확인
