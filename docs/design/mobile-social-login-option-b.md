# Mobile Social Login Option B (Segmented)

## 목표
- 이메일 로그인과 SNS 로그인을 시각적으로 분리
- 초심자 혼란 감소

## 화면 구조
- 상단: 앱 타이틀/설명
- 1영역: 이메일/비밀번호 입력 + 이메일 로그인 버튼
- 구분선: `또는 SNS로 로그인`
- 2영역: 아이콘 + 텍스트 버튼 3개(네이버/구글/카카오)
- 충돌 발생 시 별도 확인 카드 노출

## 상태 스펙
- 기본: 이메일/비밀번호 + SNS 영역 모두 노출
- 로딩: 선택한 로그인 방식만 진행 상태 표시
- 오류: 영역 내부 인라인 오류 + 상단 배너 동시 표시
- 완료: 토큰 저장 후 Main 화면 전환
- 권한 제한: provider 거부 사유 안내
- 충돌: 인라인 카드에서 `연결`, `분리 생성`, `취소`

## 반응형/다크모드
- 모바일 단일 컬럼
- 다크모드에서 구분선/보조텍스트 대비를 토큰으로 제어

## 장점
- 이메일 로그인 흐름과 SNS 흐름이 명확히 분리됨
- 상태 표시(오류/충돌)를 영역별로 표현하기 쉬움

## 리스크
- 화면 높이가 늘어나 스크롤이 필요할 수 있음
