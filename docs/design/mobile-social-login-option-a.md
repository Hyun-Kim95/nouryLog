# Mobile Social Login Option A (Button-first)

## 목표
- 로그인 진입 속도 최적화
- SNS 3종을 동일 비중 버튼으로 노출

## 화면 구조
- 상단: 앱 타이틀/설명
- 중단: 이메일/비밀번호 입력 + 이메일 로그인 버튼
- 하단: `네이버 로그인`, `구글 로그인`, `카카오 로그인` 버튼 3개
- 하단 보조: 오류/취소/재시도 안내 텍스트

## 상태 스펙
- 기본: 3개 SNS 버튼 활성
- 로딩: 선택 provider 버튼만 spinner + 전체 버튼 비활성
- 오류: 에러 배너 + 재시도 버튼
- 완료: 토큰 저장 후 Main 화면 전환
- 권한 제한: provider 정책상 거부 시 안내 텍스트 표시
- 충돌: 하단 모달로 `기존 계정 연결` / `새 SNS 계정 생성` / `취소`

## 반응형/다크모드
- 모바일 단일 컬럼
- 다크모드: 버튼 배경/텍스트 대비 4.5:1 이상 유지

## 장점
- 사용자가 SNS 로그인을 즉시 찾을 수 있음
- 진입 클릭 수가 가장 적음

## 리스크
- 로그인 화면 요소가 많아 복잡해 보일 수 있음
