# 식단 관리 디자인안 A (자체 목업)

## 개요
- 목적: 빠른 구현 전환이 가능한 실용형 UI 설계
- 기준: 현재 PRD 정책(단일 구독형, OCR 5회 무료, 배치 통계, 관리자 테이블 정책)
- 플랫폼: 모바일 앱 + 관리자 웹

## A안 디자인 원칙
- 모바일 앱은 카드/리스트 중심으로 기록 입력 흐름을 최소 탭으로 설계
- 관리자 웹은 테이블 + 상단 필터 + 하단 중앙 페이지네이션(15개)을 고정 패턴으로 사용
- 수익화 UI(결제 유도/광고 제거)는 방해 최소화, 행동 유도는 명확하게

## 모바일 앱 화면 구성
- 온보딩/로그인
  - 프로필 입력(성별, 나이, 신장, 체중, 활동량, 목표)
- 홈(오늘 요약)
  - 오늘 섭취량(칼로리/탄단지), 권장 대비 달성률
  - OCR 남은 무료 횟수 배지
- 기록 입력
  - 수동 입력 탭 / 사진 OCR 탭
  - OCR 4회 사용 시 사전 안내 배너
  - OCR 5회 소진 시 결제 유도 모달
- 통계
  - 하루/주/월 탭
  - stale 시점(`isStale=true`) 안내 배너 + 최근 집계시각
- 구독/설정
  - 무료 vs 프리미엄 상태
  - `premium_monthly` 결제/복구 버튼

## 관리자 웹 화면 구성
- 대시보드
  - KPI 카드: 신규가입, 활성사용자, 기록건수, 문의건수
  - 통계 지연 상태 카드 + `최신값 반영(강제 재집계)` 버튼
- 회원/음식/문의/공지 목록
  - 상단 필터(검색어/상태/기간)
  - 결과 테이블(기본 15개)
  - 하단 중앙 페이지네이션
  - 비활성 포함 토글

## 상태 UI 스펙 (필수)
- 기본: 정상 데이터 표시
- 로딩: 스켈레톤 또는 인라인 로더
- 빈 데이터: CTA 포함 안내
- 오류: 재시도 버튼 + 짧은 원인 안내
- 권한 제한: 접근 불가 안내 화면

## 다크모드/접근성
- 라이트/다크 토글 제공, 재방문 시 상태 유지
- 버튼/링크/입력 역할 명확화
- 대비 기준 충족(텍스트/배경/상태색)

## 구현 메모
- 컴포넌트 우선순위
  1) 기록입력/OCR 결제유도
  2) 통계/지연배너
  3) 관리자 목록 공통 테이블
