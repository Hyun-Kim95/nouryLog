# 2026-05-05 초기 의사결정 기록 (식단 관리)

## 결정 요약
1. OCR 엔진은 외부 API 사용
2. 활동량별 단백질 권장 계수표를 기본 정책으로 채택
3. 관리자 권한 레벨은 단일 레벨로 운영
4. 통계는 일 배치 집계 전략 채택
5. 삭제 정책은 soft delete 기본 + 1년 후 hard delete
6. 문의 SLA는 `2영업일 이내 답변`
7. OCR 자동입력은 회원별 무료 5회, 이후 유료 결제
8. 앱 하단 광고는 기본 노출, 결제로 광고 제거 가능
9. 통계 지연 허용치 6시간, 관리자 강제 재집계 버튼 제공
10. 감사/운영 로그 보존기간 5년
11. 결제 모델은 단일 구독형(프리미엄 월 1종)으로 운영
12. MVP 기준 가격은 `premium_monthly` 월 4,900원으로 시작

## 결정 이유
- 초기 출시 속도와 운영 단순성을 우선
- 통계 성능/비용 안정성 확보
- 정책 해석 충돌 최소화

## 영향 범위
- 앱: OCR, 통계, 권장량 계산, 과금/광고, 계정/기록 노출 정책
- 관리자: 회원/음식/문의/공지 운영 정책
- 백엔드: 배치/보존/오류 응답 규약, 과금/광고 entitlement

## 결제 모델 결정 상세
- 최종안: 단일 구독형
  - 무료: OCR 5회 + 하단 광고
  - 프리미엄 월 1종: OCR 추가 사용 + 광고 제거
  - 기준 가격: 월 4,900원
- 미채택안: 기능별 개별 구매형(OCR권/광고제거권 분리)

## 후속 검토 포인트
- OCR 벤더 품질/비용 모니터링 후 재선정 가능
- 권장량 계수표는 실제 사용자 데이터 기반 재보정
- 문의 SLA 준수율이 낮으면 인력/정책 재조정
